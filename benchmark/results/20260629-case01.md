## Problems found

1. **No feature branch (plain flow).** After `specify`, the repo is still on `main`. The
   branch is only ever created by `worktree-create.sh` (`git worktree add -b`); the plain
   non-worktree flow never creates it, yet `specify.md` and `create-feature.sh` comments say
   it does. Downstream `commit` / PR-diff against `base_branch` would compare main↔main.
   → **Needs a decision** (don't naively `git checkout -b` in `create-feature.sh` — it would
   break the later `git worktree add -b`). Options: (a) create the branch in `specify` only
   when not using worktrees, (b) drop the "creates a branch" claim from the docs for the plain
   flow. *Not auto-fixed.*

2. **Greedy feature slug.** `001-add-command-line-task-tracker-users-can` — the slug pulled raw
   words from a multi-sentence description (the 40-char cap let a 39-char sentence fragment
   through). → **Fixed** in `create-feature.sh`: added modal/stop words (`can`, `users`, …) and
   capped the slug to 5 segments. Now `add-command-line-task-tracker`. Verified by re-running
   the script directly.

3. **Command markdown is heavy (context cost).** Commands+skills total **~21.8K words (~29K
   tokens)**; `research.md` 4.2K · `implement.md` 3.4K · `plan.md` 2.4K — each loaded fresh per
   invocation. This is the "uses a lot of context" complaint, quantified. → **Mitigated** by the
   isolated-process runner (each stage pays for one command file, not the whole pipeline). A
   deeper fix (split heavy commands into a lean directive + a deferred reference section) is a
   recommendation, not done here.

## Tooling gotcha (fixed)

`setup-case.sh` preferred the `maestro` binary, which installs from **embedded resources** that
lag local `.maestro/` edits — so the fix loop would silently test stale code. Now `setup-case.sh`
**overlays** the repo's canonical `.maestro/{commands,skills,scripts,…}` after install. (To ship
a script fix to the *binary* too, run `make generate` in `cmd/maestro-cli` then reinstall.)

## Fix-loop demonstration

detect (`run-case.sh` / `DETECT_ONLY=1` → flags branch + slug) → edit
`.maestro/scripts/create-feature.sh` → overlay re-syncs it into a fresh sandbox → re-run →
slug now concise. Loop closed without rerunning the whole pipeline.
